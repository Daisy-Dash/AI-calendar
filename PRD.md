> 版本: 1.0 | 日期: 2026-06-03 | 团队: 钱嘉彤、姜雨彤、陈咏琪

## 1. 产品概述

### 1.1 产品定位

AI 日程协作者是一款 **AI 驱动的智能日程管理应用**，面向大学生群体，核心 slogan 是「**说出来，就能安排好**」。

用户通过自然语言（语音或文字）描述待办事项，AI 自动识别截止日期、优先级、预估工时，拆解为可执行的子任务，并生成可视化日历日程。团队场景下，AI 根据成员技能标签和空闲时间智能分配任务。

### 1.2 目标用户

| 维度 | 描述 |
|------|------|
| 主要用户 | 大学生（本科/研究生），18-25 岁 |
| 核心场景 | 课程项目、小组作业、毕业设计、社团活动 |
| 痛点 | 多平台信息散乱、任务不知从何下手、团队分工沟通成本高 |
| 用户特征 | 认知水平高、对复杂工具学习意愿低、喜欢温暖友好的交互 |

### 1.3 竞品分析

| 产品 | 优势 | 不足 |
|------|------|------|
| Todoist | 简洁高效 | 无 AI 拆解、中文支持弱 |
| Notion | 灵活强大 | 学习曲线陡、移动端体验差 |
| 飞书/钉钉 | 企业协作强 | 太重、不适合个人学习场景 |
| TickTick | 日历+任务一体 | AI 功能弱 |

**本产品差异化:** AI 原生 + 手绘温暖风格 + 自然语言交互 + 大学生场景定制

## 3. 非功能需求

### 3.1 性能要求
- 页面首屏加载 < 2 秒
- API 响应时间 < 500ms（P95）
- AI 解析 < 3 秒
- 支持 50+ 并发用户

### 3.2 兼容性
- 浏览器: Chrome 90+, Safari 14+, Edge 90+
- 移动端优先设计（430px 容器宽度）
- 支持 PWA 安装到 iOS/Android 桌面

### 3.3 安全性
- 密码 bcrypt 哈希存储
- JWT Token 认证，7天过期
- 用户数据隔离（只能看到自己的任务）
- API Key 服务端存储，不暴露给前端

### 3.4 可用性
- 手绘风格 UI 降低工具感
- 自然语言输入降低使用门槛
- 一键操作（首页 ✨ 悬浮按钮）
- 实时通知和自动刷新

## 4. 用户故事

### US1: 个人任务管理
> 作为一名大学生，我希望用自然语言快速记录待办事项，AI 自动帮我提取截止时间和优先级，这样我就不用手动填表了。

**验收标准:**
- 输入 "明天下午3点开会" → 自动创建日程：明天15:00，标题"开会"
- 输入 "6月15日前交毕业设计" → 创建DDL任务，截止6月15日
- 支持语音转文字输入（后续迭代）

### US2: 任务分解
> 作为一名面对大作业感到无从下手的学生，我希望 AI 帮我拆解任务为小步骤，这样我就可以一步步推进了。

**验收标准:**
- 输入任务标题+描述+天数 → AI 返回 3-5 个子任务
- 每个子任务有预估时间和优先级
- 一键确认创建所有子任务

### US3: 团队协作
> 作为一名小组组长，我希望 AI 根据成员技能自动分配任务，这样我就不用纠结谁做什么了。

**验收标准:**
- 创建群组，生成邀请码
- 成员加入后可设置技能标签
- AI 一键分配，给出分配理由
