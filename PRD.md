# AI 统筹组长 — 产品需求文档 (PRD)

> 版本: 2.0 | 日期: 2026-06-06 | 团队: 钱嘉彤、姜雨彤、陈咏琪

---

## 1. 产品概述

### 1.1 产品定位

AI 统筹组长 (AI Agent Task Assistant) 是一款专为**设计类学生、创赛团队、商科学生**等需要团队协作的人群设计的智能任务管理应用。

面对完全陌生的大型项目时，AI 不会越俎代庖直接下发任务，而是先帮你做**竞品调研、提供灵感**，等待团队**人工确认**最终方向后，再帮你把大任务**精准拆解**为可落地的个人待办。

### 1.2 目标用户

| 维度 | 描述 |
|------|------|
| 主要用户 | 设计类学生、创赛团队、商科学生等需要团队协作的群体 |
| 核心场景 | 大型课程作业、竞赛项目、跨学科团队合作 |
| 痛点 | 面对陌生任务不知从何下手、团队分工沟通成本高、缺乏调研能力、不知道最终交付物是什么样 |
| 用户特征 | 对任务领域可能完全陌生、需要零门槛的引导式体验 |

### 1.3 核心亮点

| 亮点 | 说明 |
|------|------|
| 直击真正痛点 | 解决"不知道需要什么技能"、"对最终交付物没有设想"的根本问题 |
| 零门槛消除信息差 | 内置统一 AI Agent，让整个小组在同一平台同频共振，无需各自使用外部大模型 |
| 极其克制的 AI 边界 | 故意限制 AI "出成稿"的能力，强制人类参与迭代修改，AI 是"统筹规划者"而非"替代者" |
| 符合真实创作规律 | 先搜集资料 → 再定题 → 最后分工，将 AI 能力框在前置流程中 |
| 完美的人在回路 | AI 只是探路者，最终方向必须由人类团队讨论决定 |

## 2. 功能需求

### 2.1 核心功能模块

| ID | 功能 | 描述 | 优先级 |
|----|------|------|:---:|
| F1 | 项目列表管理 | 创建/删除项目，本地持久化存储，App 重启后数据完整保留 | P0 |
| F2 | 个人技能名片 | 标签化录入专业、熟练工具，为后续智能派单打基础 | P0 |
| F3 | 破冰讨论 | 聊天式 AI 交互，AI 搜索竞品样例并提供方向建议，等待用户定夺 | P0 |
| F4 | 人工授权确认 | "确认目标并开始拆解"大按钮，硬核展示人在回路 | P0 |
| F5 | 智能任务看板 | 三泳道看板（待认领/进行中/已完成），支持手势操作 | P0 |
| F6 | 任务认领与指南 | 技能匹配雷达图 + 认领按钮 + 小白操作指南 | P0 |

### 2.2 交互要求（考核必做）

| ID | 要求 | 描述 | 状态 |
|----|------|------|:---:|
| R1 | 自定义组件 | 至少一类自定义组件（任务卡片、标签按钮等），不可全用系统原生控件 | 必做 |
| R2 | 本地数据持久化 | App 关闭重启后项目数据不丢失（localStorage） | 必做 |
| R3 | 交互手势 x2+ | 任务卡片侧滑删除、长按编辑等 | 必做 |
| R4 | 交互动画 x2+ | 页面转场切换、列表增删弹跳反馈等 | 必做 |
| R5 | 视觉反馈 | 所有操作配备视觉反馈（按钮状态变化、加载态等） | 必做 |
| R6 | 人在回路 | 明确展示人工介入机制，不可全自动黑盒运行 | 必做 |
| R7 | 前沿技术 | 至少1项（AI 长效记忆机制 — AI 记住用户专业标签并主动问候） | 必做 |

### 2.3 AI Agent 功能

| ID | 功能 | 描述 |
|----|------|------|
| A1 | 竞品调研 | AI 搜索并展示相关竞品样例与灵感方向 |
| A2 | 方向建议 | 基于用户输入的模糊需求，提供可行方向和创新点 |
| A3 | 记忆机制 | 记住用户技能标签，新项目时主动问候并基于专业给出定制建议 |
| A4 | 任务拆解 | 将确认后的大目标精准拆分为可执行的子任务 |
| A5 | 技能匹配 | 根据认领人技能标签判定任务难度，动态调整指南细致度 |
| A6 | 交付骨架 | 为每个子任务提供大纲、框架或样例，不直接生成成品 |
| A7 | 小白指南 | 针对不熟悉的领域，用通俗方式一步步教用户如何达成目标 |

## 4. 完整交互流程（Demo 演示脚本）

### Step 1: 特征录入与记忆触发
用户首次打开进入技能页选定标签。新建项目进入讨论页时，AI 直接读取记忆，依据用户专业标签主动打招呼问候。

### Step 2: 破冰启发（非直接生成）
用户输入模糊的大作业要求。AI 自动检索，返回竞品样例与方向建议，等待用户定夺。

### Step 3: 人工决策（人在回路）
团队线下讨论后，用户输入最终确定的方案。AI 弹出授权卡片，用户手动点击确认，触发转场动画。

### Step 4: 智能拆解与看板生成
页面跳转至任务看板。AI 将庞大任务转化为子节点，以列表插入动画的形式呈现在"待认领"区。

### Step 5: 人机结合派单
用户点击单张卡片进入详情。系统显示难度匹配度，用户手动点击认领，随后解锁针对该任务的专属实操指南。

### Step 6: 重启验证（持久化检测）
将 App 后台杀掉并重新打开，进入首页点击项目，验证任务看板和认领状态原样保留。
