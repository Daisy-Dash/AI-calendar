## 1. 设计理念

### 1.1 核心设计原则

| 原则 | 说明 | 体现 |
|------|------|------|
| **温暖亲切** | 降低工具感，让用户感到舒适而非压力 | 手绘风格、暖色系、emoji 点缀 |
| **说即所得** | 自然语言是最自然的交互方式 | 语音/文字输入 → AI 自动结构化 |
| **一眼即知** | 信息层级清晰，关键信息突出 | 颜色编码、进度条、DDL 倒计时 |
| **渐进呈现** | 先展示核心信息，需要时再展开详情 | 日历日点击展开、任务卡片折叠 |
| **容错友好** | 用户不精确的描述也能被理解 | 模糊日期解析、智能回退 |

### 1.2 视觉关键词

**手绘** · **暖色调** · **圆润** · **轻盈** · **趣味**

## 6. 交互模式

### 6.1 自然语言输入（核心交互）

**场景**: 用户在任何时候想快速记录一件事

**流程**:
1. 点击首页 ✨ 悬浮按钮
2. 弹出输入面板，展示示例提示词
3. 用户输入（或点击示例）
4. 点击「解析」→ AI 识别日期/时间/标题
5. 展示解析结果预览
6. 用户确认 → 创建日程/任务 → 跳转对应页面

**支持的输入格式**:
```
✅ 明天下午3点开会          → 日期:明天, 时间:15:00, 标题:开会
✅ 后天交作业               → 日期:后天, 标题:交作业
✅ 6月10日聚餐              → 日期:6/10, 标题:聚餐, 颜色:橙
✅ 今晚8点运动              → 日期:今天, 时间:20:00, 标题:运动
✅ 下周三下午2点组会         → 日期:下周三, 时间:14:00, 标题:组会
✅ 2026-06-15 提交论文      → 日期:6/15, 标题:提交论文
```

### 6.2 日历操作

- **月视图**: 点击日期 → 下方展示当日日程列表
- **日程点**: 日历格子上彩色圆点 = 该日有日程
- **本周条**: 横向展示本周每天日程摘要
- **快速添加**: 选中日期 → 点「+」→ 弹窗创建

### 6.3 进度追踪

- **快速标记**: 任务卡片上的 25/50/75/100 百分比按钮
- **一键完成**: 点击空心圆 → 变为绿勾 ✓
- **撤销完成**: 点击绿勾 → 恢复未完成
- **进度条动画**: 0.5s transition

### 6.4 团队协作

- **创建群组**: 输入名称 → 自动生成6位邀请码
- **加入群组**: 输入邀请码 → 确认加入
- **AI 分配**: 点击按钮 → AI 分析成员技能 → 生成分配方案
- **成员管理**: 群主可移除成员、查看完成率

### 6.5 微交互

| 交互 | 效果 |
|------|------|
| 按钮 hover | 轻微上浮 + 阴影加深 |
| 按钮 active | 按下位移 + 阴影减弱 |
| 卡片 active | scale(0.98) 按压反馈 |
| 页面进入 | fadeInUp 0.3s |
| 加载中 | 旋转动画 + 省略号闪烁 |
| 通知 Toast | 顶部滑入, 3s后消失 |
| 下拉刷新 | 下拉指示器 + 旋转加载 |
| 开关 toggle | 平滑滑动 0.2s |

## 7. 响应式策略

### 7.1 断点定义

```
📱 手机 (默认)  < 430px    全宽布局，底部导航
📱 大屏手机      430-768px  居中容器(430px)，两侧留白
💻 平板/桌面     > 768px   居中容器(430px)，模拟手机预览
```

### 7.2 PWA 适配

```
manifest.json:
  display: standalone         全屏独立窗口
  orientation: portrait       竖屏优先
  theme_color: #FF9F43        状态栏橙色
  background_color: #FFF8F0   启动背景暖色
```
